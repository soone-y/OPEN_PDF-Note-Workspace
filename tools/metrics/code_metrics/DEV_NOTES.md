# code_metrics 開発メモ

## 目的

このツールは、規模の変化を安全に観測するためのローカル専用ツールです。
外部通信は行わず、集計対象のファイルを書き換えません。

## 初期実装の範囲

確実に取りやすい指標だけを対象にしています。

- ディレクトリ構造の要約
- ファイル数
- ファイルサイズ
- 行数
- 文字数
- 拡張子別内訳
- 関数定義数の概算
- 型定義数の概算
- 依存参照数
- 空行 / コメント行 / コード行
- TODO 系コメント数
- 最長行長
- 近似の関数宣言数 / 参照数
- 近似の変数宣言数 / 参照数
- 近似の未使用候補数

## 方針

- `own` と `all` を切り替えられるようにする
- `own` は自作コードの実態把握に寄せる
- バイナリやサードパーティ依存の偏りで読みづらくならないように既定除外を入れる
- JSON 出力で後続の可視化や差分比較につなげやすくする

## 現状の関数数カウント

- Python は `def` / `async def` をカウント
- C/C++ は関数定義らしい行を正規表現で概算
- 宣言だけのプロトタイプや `if` / `for` / `while` などは除外
- 型定義は Python の `class`、C/C++ の `class` / `struct` / `enum` を概算
- 依存参照は Python の `import` / `from ... import`、C/C++ の `#include` を集計

## 精度注意の近似解析

- `approx_` で始まる項目は、傾向を見るための参考値
- C/C++ はコメント除去後に正規表現ベースで関数名・変数名候補を抽出
- Python は `def` と代入文、`for` 変数などを候補として抽出
- 参照回数は識別子出現回数から近似
- 宣言数以上しか参照がない識別子を未使用候補として扱う
